# 对话系统

对话系统是 AIPP 的核心功能模块，提供完整的 AI 对话交互体验。

---

## 对话创建与管理

### 创建新对话
用户可以通过多种方式创建新对话：
- 直接在 ChatUI 窗口中点击「新对话」按钮创建
- 通过 Ask 窗口提问时自动创建新对话
- 使用 `create_conversation_with_messages` API 创建带初始消息的对话
- 创建时可指定对话名称、系统提示词、用户消息等参数
- 对话会关联到选定的助手配置和模型配置

### 对话列表展示
- 分页加载对话列表，支持分页参数（page、page_size）
- 显示对话标题、助手名称、创建时间等信息
- 使用助手名称缓存提高列表加载性能
- 通过 `conversation_created`、`conversation_deleted` 事件实时更新列表

### 对话删除
- 支持删除整个对话及其所有消息
- 删除后发送 `conversation_deleted` 事件通知前端更新
- 级联删除对话相关的所有消息和附件

### 对话分支（Fork）
- 支持从指定消息位置分叉新对话
- 保留原对话历史到分叉点
- 自动在标题后添加"版本N"标识
- 复制消息附件
- 同时复制的消息更新创建时间

### 对话搜索
- 支持按关键词搜索对话
- 搜索范围包括：对话标题、对话总结、消息内容
- 返回匹配结果带高亮片段（snippet）
- 标注命中类型（title/summary/message）
- 支持 limit 和 offset 分页

---

## 消息版本化与分支

### 消息重新生成
- 支持对 AI 响应消息重新生成
- 每次重新生成创建新的消息版本
- 使用 `generation_group_id` 标识同一次生成的消息组
- 使用 `parent_group_id` 标识父版本组，形成版本树

### 版本分页浏览
- 前端 `useMessageGroups.ts` 处理版本分组逻辑
- 支持在不同版本间切换浏览
- 版本分页组件显示当前版本/总版本数
- 提供上一版本/下一版本导航按钮

### 消息排序规则
- 按创建时间排序
- 同一 generation_group_id 中 reasoning 消息排在 response 之前
- 消息类型优先级：system > user > reasoning > response > assistant

## 流式响应

### 流式聊天实现
- 使用 genai 客户端的 ChatStreamEvent 流式接口
- 支持流式和非流式两种模式
- 流式响应实时更新消息内容

### 实时 UI 更新
- 通过 `message_update` 事件推送内容更新
- 首 Token 时间（TTFT）记录和展示
- 逐步追加内容到消息
- 实时更新 token 计数

### 响应取消功能
- 使用 CancellationToken 实现取消
- `cancel_ai` 命令中止当前生成
- 清理会话状态和任务
- 支持 ACP 会话任务中止

### 错误处理与重试
- 自动重试机制，可配置重试次数
- 指数退避重试延迟计算
- HTTP 错误详情提取（状态码、响应体、request_id）
- 友好的错误信息展示给用户

## 对话标题生成

### 自动生成标题
- `generate_conversation_title` API 自动生成
- 使用配置的标题生成模型
- 基于对话内容智能生成简洁标题

### 手动编辑标题
- `update_conversation` API 更新标题
- 发送 `title_change` 事件通知
- 用户可随时修改对话标题

---

## 对话总结

### 总结生成
- `summary_conversation` API 生成对话总结
- 可配置专用总结模型
- 总结结果持久化存储

---

## 消息编辑

### 编辑用户消息
- `update_message_content` API 更新消息内容
- 支持编辑用户消息后重新生成 AI 响应

### 编辑 AI 响应
- 支持手动修改 AI 生成的内容
- 仅允许编辑 assistant 类型消息
- 更新完成时间标记

---

## 附件功能

### 文件附件上传
- 支持文件拖拽上传（`useFileDropHandler` 提供拖拽事件处理和视觉反馈）
- 附件与消息关联存储
- 支持多附件
- 拖拽状态跟踪和浏览器默认行为拦截
- 附件通过 SHA256 哈希去重，避免重复上传

### 图片附件支持
- 图片附件预览
- 多模态输入支持
- 自动检测文件类型（Image/Text/PDF/Word/PowerPoint/Excel）

### 附件操作
- `add_attachment` 通过 URL 或内容添加附件
- `open_attachment_with_default_app` 使用系统默认应用打开附件
- 支持直接内容提交和文件路径引用两种方式

### 选中文本引用
- `useTextSelection` Hook 捕获系统级选中文本
- 通过 Bang 命令 `!selected_text` 将选中文本插入到对话中
- 支持跨窗口文本捕获

## Bang 命令系统

### 概述
- Bang 是动态命令替换系统，用户在输入框中输入 `!` 或 `！` 触发
- 类似 DuckDuckGo Bangs 或 Shell Bang 命令
- 输入 Bang 命令后，消息发送时由 `TemplateEngine` 解析并替换为实际内容

### 内置 Bang 命令
| 命令 | 快捷方式 | 用途 |
|------|---------|------|
| `!current_date` | `!cd` | 插入当前日期（YYYY-MM-DD） |
| `!current_time` | `!ct` | 插入当前时间（HH:MM:SS） |
| `!selected_text` | `!s` | 插入当前选中的文本 |
| `!web(URL)` | `!w` | 获取网页内容（HTML 格式） |
| `!web_to_markdown(URL)` | `!wm` | 获取网页内容（Markdown 格式） |
| `!file(path)` | - | 读取文件内容 |
| `!sub_start(text, count)` | - | 截取文本前 N 个字符 |

### 补全交互
- 输入 `!` 或 `！` 时自动弹出 `BangCompletionList` 补全下拉框
- 支持按名称和快捷方式过滤
- 键盘导航（↑↓ 箭头）、Tab/Enter 选择、Esc 关闭
- 选中后自动补全命令模板，光标定位到参数位置

### 模板引擎
- 后端 `template_engine/mod.rs` 负责解析和执行 Bang 命令
- 使用正则匹配 `[!！](\w+)(\(...)?)`  提取命令
- 支持 Text、Image、Audio 三种输出类型（Image/Audio 预留扩展）
- 同时支持中文感叹号 `！` 和英文感叹号 `!`

## 输入补全系统

### Bang 命令补全
- `BangCompletionList.tsx` - 输入 `!` 触发
- 显示命令名称、快捷方式和描述
- 快速插入动态内容

### 助手补全
- `AssistantCompletionList.tsx` - 输入 `@` 触发
- 支持拼音全拼、空格拼音和首字母匹配搜索助手（如 `shitu` → 识图，`stcs` → 识图测试）
- 匹配字符自动高亮
- 快速切换或引用助手

### 制品补全
- `ArtifactCompletionList.tsx` - 输入特定触发符时显示
- 显示已保存的制品列表
- 同样支持拼音搜索过滤
- Tab 键补全、Enter 键打开制品

## 思考过程显示

### Reasoning 消息
- 支持模型思考过程单独存储为 reasoning 类型消息
- 与 response 消息同属一个 generation_group_id
- 在 UI 中可展开/收起显示
- 排序时保证 reasoning 排在 response 之前

## 性能优化

### 性能监控
- `get_conversation_with_messages` 分阶段耗时统计
- 查询 conversation、查询 messages、构建 MessageDetail、处理版本管理分别计时
- 输出 PERF 日志便于排查性能问题

### 名称缓存
- `NameCacheState` 缓存助手名称
- 避免频繁查询数据库
- 提升列表加载速度

## 对话导出

### 多格式导出
- 支持将对话导出为 **Markdown**（`.md`）、**PDF**（`.pdf`）、**PNG**（`.png`）三种格式
- Markdown 导出保留完整消息格式，包含角色标记和时间信息
- PDF 导出使用 A4 智能分页，不会在消息中间截断
- PNG 导出通过 html2canvas 渲染，支持可配置宽度

### 导出选项
- 可选择包含/排除工具调用、调用参数、调用结果
- 可选择包含/排除系统提示词和思考过程
- 按对话分支过滤消息
- 暗色/浅色模式自适应渲染

### 导出对话框
- `ConversationExportDialog.tsx` 导出配置对话框
- `ConversationExportRenderer.tsx` 导出渲染组件
- 一键复制和文件保存

---

## 对话搜索

### 全文搜索
- `conversationSearchService.ts` 提供全文搜索服务
- 搜索范围覆盖对话标题、对话总结、消息内容
- 返回匹配结果带高亮片段（snippet）
- 标注命中类型（title/summary/message）

### 搜索界面
- `ConversationSearchDialog.tsx` 搜索对话框
- 支持 limit 和 offset 分页
- 搜索结果点击跳转到对应对话
