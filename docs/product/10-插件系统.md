# 插件系统

插件系统允许开发者以最小侵入的方式扩展 AIPP 的功能，支持新增助手类型、注入界面组件、扩展后台逻辑等。

---

## 插件类型

### AssistantType (1)
- **助手类型插件** - 在「个人助手配置」页面注册并渲染新的助手类型
- 当前已实现，可直接使用
- 用于扩展不同类型的 AI 助手（如代码生成、学术写作等）

### InterfaceType (2)
- **界面插件** - 渲染独立窗口或嵌入式 UI
- 预留接口，即将支持
- 用于添加自定义 UI 组件和窗口

### ApplicationType (3)
- **应用插件** - 脱离 UI、提供后台能力
- 预留接口，即将支持
- 用于后台服务和定时任务等

## 插件生命周期

### onPluginLoad
- 插件加载完成后调用
- 可用于初始化插件状态
- 传入 `systemApi` 系统 API 对象

### config
- 返回插件配置信息
- 声明插件名称、类型等元数据
- 框架据此判断插件类型和能力

### onAssistantTypeInit
- 助手类型初始化时调用
- 注册助手类型和表单字段
- 传入 `assistantTypeApi` API 对象

### onAssistantTypeSelect
- 用户在下拉框选中该助手类型时触发
- 可设置默认值、隐藏字段等
- 传入 `assistantTypeApi` API 对象

### onAssistantTypeRun
- 点击「运行」后触发
- 实现助手的具体运行逻辑
- 传入 `assistantRunApi` API 对象

### renderComponent
- 界面插件渲染组件
- 返回 React 组件
- 用于界面插件类型

## 助手类型插件开发

### 助手类型注册
- `assistantTypeApi.typeRegist()` 注册新助手类型
- 指定类型 ID、名称、插件实例
- 类型 ID 避免与现有重复

### 动态表单字段添加
- `assistantTypeApi.addField()` 添加表单字段
- 配置字段名称、标签、类型
- 设置字段位置（header/body）和提示信息

### 字段隐藏/显示控制
- `assistantTypeApi.hideField()` 隐藏字段
- `assistantTypeApi.showField()` 显示字段
- 可在 `onAssistantTypeSelect` 中动态控制

### 字段默认值设置
- `assistantTypeApi.forceFieldValue()` 强制设置字段值
- 常用于助手类型切换时设置默认参数

### 子任务注册
- `assistantTypeApi.subTaskRegist()` 注册子任务
- 配置子任务编码、名称、描述、系统提示词
- 可选配置 `iconComponent` 图标组件

### 助手运行逻辑实现
- `assistantRunApi.getUserInput()` 获取用户输入
- `assistantRunApi.askAI()` 调用 AI
- `assistantRunApi.askAssistant()` 调用其他助手
- `assistantRunApi.appendAiResponse()` 追加 AI 响应

## 插件加载机制

> **注意**: 插件系统的核心逻辑在前端实现（`src/plugins/`），后端 `src-tauri/src/plugin/mod.rs` 为空模块，仅提供数据库存储支持（`plugin_db.rs`）。插件的加载、生命周期管理和 API 注入均由前端 JavaScript 完成。

### 插件目录结构
```
<AppDataDir>/plugin/<plugin_code>/
  └─ dist/
     └─ main.js  # 产物入口，必须导出全局插件类
```

### 插件脚本动态加载
- ConfigWindow 和 PluginWindow 运行时注入 `<script>` 标签
- 脚本路径指向插件的 `main.js`
- Tauri 提供跨平台自动定位 AppDataDir

### 全局插件类挂载
- 插件类必须挂载到 `window` 全局对象
- 例如：`window.SamplePlugin = MyPlugin`
- 全局名称需与加载列表保持一致

### 插件 API 注入
- 加载完成后框架查找插件类
- 实例化插件并触发生命周期函数
- 注入相应的 API 对象（systemApi、assistantTypeApi 等）

## 子任务图标

### 图标组件注册
- 注册子任务时可传入 `iconComponent`
- 支持两种类型：
  - `React.ReactNode` - 直接渲染的节点
  - `React.ComponentType` - 组件类型（如 lucide-react 图标）

### 图标渲染
- 框架按 `code` 注册到运行期前端注册表
- UI 在子任务选择/详情处显示图标
- lucide-react 图标支持 size、className 等属性

### 示例
```tsx
import { Wrench } from 'lucide-react';

// 使用组件类型
iconComponent: Wrench

// 或使用直接渲染的节点
iconComponent: <Wrench className="h-4 w-4" />
```
