# 数据存储

数据存储模块使用多个独立的 SQLite 数据库实现所有数据的本地存储，确保用户数据完全在本地，不上传云端。

---

## SQLite 本地存储

### 所有数据本地存储，不上传云端
- 本地优先的设计理念
- 所有数据存储在用户设备上
- 无云端同步功能
- 保护用户隐私

### 分离的 SQLite 数据库
- 使用多个独立的 SQLite 数据库文件
- 按功能域分离数据库，便于管理和维护
- rusqlite 库提供 Rust 绑定
- 事务支持，数据一致性保证

### rusqlite ORM
- `db/mod.rs` 数据库主模块
- Repository 模式封装 CRUD 操作
- 类型安全的数据访问
- 连接池管理

### 数据库迁移管理
- 迁移脚本版本管理
- 自动迁移检测和执行
- 向下兼容性保证
- 迁移失败回滚

## 主要数据表说明

### conversation - 对话记录
- 对话基本信息（ID、名称、创建时间等）
- 关联的助手 ID
- 对话列表查询
- 对话创建、更新、删除

### message - 消息记录（支持版本化）
- 消息内容和类型
- generation_group_id 和 parent_group_id 版本管理
- Token 计数（input/output/total）
- 时间戳（开始、完成、首 Token 时间）
- 工具调用 JSON
- 消息附件关联

### message_attachment - 消息附件
- 附件关联的消息 ID
- 附件类型（Image, Text, PDF, Word, PowerPoint, Excel）
- 附件元数据
- 文件路径或内容

### assistant - 助手配置
- 助手名称和描述
- 助手类型（0=普通对话, 1=多模型对比, 2=工作流, 3=展示, 4=ACP 助手）
- 系统提示词
- 助手配置 JSON
- 助手模型关联

### llm_provider - LLM 提供商
- 提供商名称和类型
- API Key 和 Base URL
- 提供商配置
- 启用/禁用状态

### llm_model - LLM 模型
- 模型名称和代码
- 关联的提供商 ID
- 模型参数（max_tokens、temperature 等）
- 模型配置 JSON
- 启用/禁用状态

### mcp_server - MCP 服务器配置
- 服务器名称
- 命令和工作目录
- 环境变量和附加参数
- 服务器配置 JSON

### conversation_todo - 对话 Todo
- Todo 任务关联对话
- Todo 内容和状态
- 任务创建和更新

## 数据隐私

### 本地存储，无云端同步
- 所有数据存储在本地 SQLite 文件
- 无任何云端同步功能
- 用户完全控制数据
- 数据不离开用户设备

### 无遥测功能
- 不收集用户使用数据
- 不发送任何分析数据
- 完全透明的代码
- 隐私优先设计

### 用户数据完全掌控
- 用户可以备份数据库文件
- 用户可以删除所有数据
- 数据格式开放
- 无数据锁定

---

## 数据库模块架构

### Repository 模式
- 每个实体有独立的 Repository
- ConversationRepository、MessageRepository 等
- 统一的 CRUD 接口
- 类型安全的数据访问

### 测试隔离
- 测试使用内存数据库
- `Connection::open_in_memory()` 创建测试数据库
- 测试之间完全隔离
- 不影响真实数据
