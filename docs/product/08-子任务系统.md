# 子任务系统

> ⚠️ **注意**: 子任务系统后续计划删除，请不要依赖此功能。当前代码仍完整运行中（`sub_task_db.rs`、`sub_task_api.rs`），但不建议新功能依赖此模块。

子任务系统允许将复杂任务拆分为多个子任务，每个子任务有独立的系统提示词和配置，支持 MCP 工具循环调用。

---

## 子任务定义

### 子任务注册
- `sub_task_api.rs` 子任务 API
- `register_sub_task_definition` / `sub_task_regist` 注册子任务
- 子任务编码（code）唯一标识
- 子任务名称和描述

### 子任务名称与描述
- 子任务名称：简短易读的名称
- 子任务描述：详细说明子任务功能
- 支持多语言（预留）

### 子任务系统提示词
- 每个子任务有独立的系统提示词
- 系统提示词定义子任务的行为
- 提示词支持模板变量

### 子任务图标配置
- 注册子任务时可传入 `iconComponent`
- 支持 lucide-react 图标组件
- 支持直接渲染 ReactNode
- 图标按子任务 code 注册

### 子任务 CRUD
- `list_sub_task_definitions` - 列出子任务定义
- `get_sub_task_definition` - 获取子任务定义
- `update_sub_task_definition` - 更新子任务定义
- `delete_sub_task_definition` - 删除子任务定义

---

## 子任务执行

### 子任务执行创建
- `create_sub_task_execution` - 创建子任务执行
- `list_sub_task_executions` - 列出子任务执行
- `get_sub_task_execution_detail` - 获取子任务执行详情
- `get_sub_task_execution_detail_for_ui` - 获取子任务执行详情(UI)

### 子任务运行
- `run_sub_task_sync` - 同步运行子任务
- `run_sub_task_with_mcp_loop` - 带 MCP 循环的子任务运行
- 使用子任务的系统提示词
- 子任务特定配置
- 独立的对话上下文

### 子任务 MCP 调用
- `get_sub_task_mcp_calls_for_ui` - 获取子任务 MCP 调用(UI)
- 子任务执行过程中的 MCP 工具调用
- 工具调用历史记录

### 子任务取消
- `cancel_sub_task_execution` - 取消子任务执行
- `cancel_sub_task_execution_for_ui` - 取消子任务执行(UI)

### 子任务结果处理
- 子任务执行结果
- 结果合并到主对话
- 子任务状态记录

---

## 子任务图标

### lucide-react 图标组件支持
- 支持 lucide-react 图标组件类型
- 如 `Wrench`、`Check` 等
- 框架自动设置合适的 size 和 className
- 图标主题适配

### ReactNode 直接渲染
- 支持直接传入 ReactNode
- 例如 `<Wrench className="h-4 w-4" />`
- 完全自定义样式
- 灵活的图标展示

### 图标注册与管理
- 图标按子任务 code 注册到运行期前端注册表
- 后续 UI 在子任务选择/详情处显示图标
- 仅前端注册，无需后端 API 或数据库调整
- 图标注册表在内存中管理
