# 多模型支持

多模型支持模块允许用户连接和管理多种大语言模型提供商，通过统一的接口进行对话。使用rust-genai来进行支撑该功能的。

---

## LLM 提供商管理

### 提供商配置
- 支持添加多种 LLM 提供商（OpenAI、Anthropic、Google 等主流提供商）
- 每个提供商可配置 API Key、Base URL、API 版本等参数
- 提供商配置持久化到 SQLite 数据库
- 支持多提供商同时启用

### 提供商 CRUD
- `add_llm_provider` - 添加新的 LLM 提供商
- `get_llm_providers` - 获取所有已配置的提供商列表
- `get_filtered_providers` - 获取过滤后的提供商列表
- `update_llm_provider` - 更新提供商配置
- `delete_llm_provider` - 删除提供商
- `export_llm_provider` - 导出 LLM 提供商
- `import_llm_provider` - 导入 LLM 提供商

### 提供商配置
- `get_llm_provider_config` - 获取 LLM 提供商配置
- `update_llm_provider_config` - 更新 LLM 提供商配置

---

## 模型配置

### 模型管理
- 支持为每个提供商添加多个模型
- 每个模型可配置独立参数：
  - `max_tokens` - 最大 Token 数
  - `temperature` - 温度参数
  - `top_p` - Top-p 采样
  - 其他模型特定参数

### 模型分类
- 按提供商分组显示
- 模型与提供商关联存储
- 支持启用/禁用单个模型

### 模型 CRUD
- `add_llm_model` - 添加新模型
- `get_llm_models` - 获取模型列表
- `update_llm_model` - 更新模型配置
- `delete_llm_model` - 删除模型
- `get_models_for_select` - 获取用于选择的模型
- `get_filtered_models_for_select` - 获取过滤后的用于选择的模型

### 模型列表获取
- `fetch_model_list` - 从提供商获取模型列表
- `preview_model_list` - 预览模型列表
- `update_selected_models` - 更新选中的模型

---

## 模型切换

### 对话内切换
- 支持在对话中切换模型
- 切换后新消息使用新模型
- 历史消息保留原模型信息

### 对话独立配置
- 不同对话可以使用不同模型
- 对话创建时绑定当前选择的模型

### 无模型回退策略
- 当配置的模型不存在时，禁止自动回退到其他模型
- 直接返回错误信息，在界面上提示用户检查配置
- 确保用户明确知道配置问题所在

## genai 客户端集成

### 自定义 genai 客户端
- 基于自定义 fork 的 genai 库
- 支持多种模型提供商统一封装
- 流式聊天支持
- 工具调用支持

### ChatRequest 构建
- 系统提示词合并
- 消息历史构建
- 工具配置合并
- ChatOptions 构建（temperature、max_tokens 等）

### 流式响应处理
- ChatStreamEvent 事件流
- Token 计数
- 工具调用检测
- 错误处理与重试

---

## 配置合并

### 配置优先级
- 助手模型配置优先级高于提供商默认配置
- 支持按场景配置覆盖
- 最终合并后的配置传递给 genai 客户端

### 配置验证
- 验证模型是否存在
- 验证提供商配置有效性
- 无回退，缺失直接报错

---

## Provider 分享

### 加密分享
- 支持将 LLM 提供商配置导出为加密分享码
- 使用 AES-256-GCM 加密保护 API Key 等敏感信息
- 分享码通过 gzip 压缩 + Base64 编码生成
- 导入时需要输入密码解密

### 分享流程
- `export_llm_provider` 导出提供商配置为分享码
- `import_llm_provider` 通过分享码导入提供商配置
- `ShareDialog.tsx` 分享对话框 UI
- 一键复制分享码到剪贴板
