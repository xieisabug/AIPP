# AIPP 插件系统设计文档（阅读顺序）

你可以按下面顺序看，越往后越落地：

1. **`plugin-kernel-01-四层架构白话.md`**  
   先用通俗语言理解“四层”到底是什么，避免一开始陷进技术细节。
2. **`plugin-kernel-02-和当前AIPP如何结合.md`**  
   再看“不是推倒重来”，而是怎么把新设计接到你现在的 AIPP 上。
3. **`plugin-kernel-03-详细实施计划.md`**  
   再看分阶段怎么做，每阶段产出什么、验收什么。
4. **`plugin-kernel-04-四个玩法落地方案.md`**  
   最后看 AI竞技场 / 多模型剧场 / AI伙伴养成 / 成就模块分别怎么落地。

## 你看完后应该得到的结论

- 你会知道这四层不是“为了复杂而复杂”，而是为了让插件系统**可扩展、可控、可持续**。
- 你会知道当前 AIPP 哪些能复用、哪些必须补。
- 你会有一条清晰的落地路径，而不是“大改一波”。
