# AIPP 插件系统设计文档（阅读顺序）

你可以按下面顺序看，越往后越落地：

1. **`plugin-kernel-01-四层架构白话.md`**  
   先用通俗语言理解“四层”到底是什么，避免一开始陷进技术细节。
2. **`plugin-kernel-02-和当前AIPP如何结合.md`**  
   再看“不是推倒重来”，而是怎么把新设计接到你现在的 AIPP 上。
3. **`plugin-kernel-03-详细实施计划.md`**  
   再看分阶段怎么做，每阶段产出什么、验收什么。
4. **`plugin-kernel-04-四个玩法落地方案.md`**  
   再看 AI竞技场 / 多模型剧场 / AI伙伴养成 / 成就模块分别怎么落地。
5. **`plugin-kernel-05-技术实现细节.md`** ⭐  
   最关键的一份——包含完整代码示例、API 规格、界面挂载方式和迁移路径。
6. **`plugin-kernel-06-插件类型系统设计.md`** ⭐ NEW  
   V2 类型内核收敛版：先稳定 `assistant/ui/worker` 三类能力与生命周期。
7. **`plugin-kernel-07-基于底层扩展路线.md`** ⭐ NEW  
   专门讲“后续怎么从底层扩出 Theme / Markdown / Tool 等能力”，含阶段与验收。

---

## 速读路径（10 分钟）

如果你时间少，直接看：

- 01 的「一图看懂」
- 02 的「当前短板清单」和「与四个玩法的映射」
- 03 的「分阶段计划总览表」
- 04 的每个玩法里的「MVP（第一版）」小节
- **05 的「插件如何改变界面」和「四个玩法的插件设计详解」**（最具体的部分）
- **06 的「类型收敛原则」和「Capability 最小集」**（先把底层做稳）
- **07 的「扩展方法论」和「Theme/Markdown 落地路径」**（再做可持续扩展）

## 你看完后应该得到的结论

- 你会知道这四层不是“为了复杂而复杂”，而是为了让插件系统**可扩展、可控、可持续**。
- 你会知道当前 AIPP 哪些能复用、哪些必须补。
- 你会有一条清晰的落地路径，而不是“大改一波”。
